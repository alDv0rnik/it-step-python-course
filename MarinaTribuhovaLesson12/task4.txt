Задание 4 (I). Запустите модуль modules/mod_a.py. Проверьте его результат. Объясните, почему
это происходит.
Попробуйте изменить х на список [1,2,3]. Объясните результат.
Попробуйте изменить импорт на from х import *̇ где x - имена модулей. Объясните результ

При запуске mod_a.py переменная x будет хранить ссылку на объект 1000, при последовательных импортах в mod_b.py при импорте mod_c
 в переменной сохранится ссылка на объект 1000 вместо ссылки на объект 6. Такая возможность сохраняется за счет явного обращения 
mod_c.x = 1000. Т.к. импорт модуля выполняется один раз, следовательно импорт mod_c.py выполниться только внутри mod_b.py, вызванном через 
from mod_b import в модуле mod_a.py. Что легко проследить при добавлении в mod_c.py скрипта print("Hello mod_c!!!").На вывод получим: 
Hello mod_c!!!
1000

При изменинии х на список [1,2,3], также вывод 1000, переменная x хранит уже ссылку не на объект [1, 2, 3], а на новый объект 1000
при предложенной последовательности импортов.Такая возможность сохраняется за счет явного обращения mod_c.x = 1000.Т.к. импорт модуля 
выполняется один раз, следовательно импорт mod_c.py выполниться только внутри mod_b.py,вызванном через from mod_b import в модуле mod_a.py.
Что легко проследить добавив в mod_c.py скрипт print("Hello mod_c!!!").На вывод получим: 
Hello mod_c!!!
1000

При изменеии местами import первым срабатывает импорт from mod_c import x в mod_a.py.Переменная x будет хранить ссылку на объект 6 или [1,2,3].
Что также объясняется вызовом модуля один раз!На вывод получим   
Hello mod_c!!!
6
или
Hello mod_c!!!
[1, 2, 3]
